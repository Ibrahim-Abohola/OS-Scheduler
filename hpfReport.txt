DataStructure Used: Priority Queue

Explain:
At each clock the scheduler blocks on a semaphore until the Process Generator finishes sending all processes for that clock After that all processes for this clock are received so the scheduler begins to handle them.
It receives all incoming processes and if there is a running process and a higher-priority process arrives it preempts the running process and stops it and saves its state then all received processes alongside with the stopped one are added to the Priority Queue.
After that if there is no running process the scheduler extracts the highest-priority process from the queue if the extracted process wasn’t stopped before it forks and starts a new process otherwise it resumes the paused process.

Time taken: about 3 days
